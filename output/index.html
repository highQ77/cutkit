歡迎使用 CutKit 開源免費切版工具
一直以來切版的職責歸屬是模凌兩可的，而跟切版工作最關聯的職位是網頁設計師與網站前端工程師，
在實際工作中切版的工作會被踢提皮球；而為了讓切版開發更為有趣，筆者開發 CutKit 切版輔助工具，
讓視覺介面開發簡單且提高生產力。

若是你學會了 CutKit，你的開發速度將會大幅提升。筆者建議 CutKit 選配 TailwindCSS，
在 RWD 處理較為輕鬆 --- 本專案以 TailwindCSS 為基底開發。

> <切版不是一份討喜的工作原因可能為>：
+ 對視覺設計師而言只想設計圖案而不想碰程式
+ 初學者想上手 CSS 但需要記得的 CSS 屬性指令太多，不知如何開始
+ CSS 學習 cp 值不高，只能用在網頁介面的開發上
+ CSS selector 規則太多太複雜，有時選不到相應的物件
+ 有時設定的 CSS 指令總是無法達到想要的效果
+ SCSS / SASS / LESS 等太多預處理器有選擇障礙
+ HTML 遇到重複的結構但不同的資料在切版時要大量複製貼上，然後找修改的資料時眼睛都看花了
+ 常態性 CSS 視覺的調整頻率，例如客戶要求的調整

> <CutKit 解決了什麼問題>？
+ 提高切版創作動力，讓你愛上切版
+ 建立覆用元件模組讓開發更流暢，無需大量 copy / paste / modify 程式碼
+ 元件模組可以分享給不同的切版師
+ 配合 TailwindCSS 搭建介面，亦可依照個人需求選擇不同 CSS 框架，譬如 Bootstrap
+ 模組錯誤程式片段修正只要一個地方，若是大量使用此模組修改不費心
+ wireframe ( 線框圖 ) 完成後進行 mockup ( 視覺切版搞 )，如果夠厲害可直接邊想邊切版生成
+ 製作切好版的 wireframe 加上一些互動流程機制即為 prototype ( mockup + 簡單互動程式 ex: 開關對話框 / 跳轉頁面 )

安裝
安裝指令
git clone xxx/cutkit.git    <!--下載專案-->

cd cutkit                   <!--進到 cutkit 專案資料夾-->

npm i                       <!--下載相依套件-->

開發流程
流程介紹
個人認為合理的專案開發流程

<!-- 下面 5 個步驟為一個小的專案功能流程，一個專案有多個流程，譬如流程可為使用者註冊登入流程、發送電子報流程等等，循環 5 個步驟一步一步解決各個流程直至完成所有流程 -->

1. 繪製 wireframe 草稿圖 <draw.io>，畫面由線條與框框組成，其用意是規劃出畫面的陳設與功能，不在意畫面是否美觀，但是功能都需要確認、UI位置配置正確
2. 完成 wireframe 後，開始製作 mockup，通常是 UIUX / 網頁設計師的工作，將視覺介面繪製出來 <figma>
3. 當 mockup 畫面確立沒問題，下一個階段即切版，<歡迎使用 CutKit> 進行切版設計
4. 切版完成後，開始製作 prototype，這邊的 prototype 是指 show / hide popup 與跳轉頁面的簡單互動機制
5. 測試、優化與調整


<!-- CutKit 操作步驟 -->

1. page 資料夾內建立自己的 xxx.cut
2. xxx.cut 透過 component/xxx 與 compose/xxx 組成 xxx.cut 畫面
3. 加入 event 機制完成 prototype 製作
4. 確認沒問題交付整個專案給前端工程師
5. 前端工程師依照專案 component / compose / page 裡的元件刻製 react 或 vue 版本的元件 

(若自動生成 react 或 vue 元件或許可以省下大家很多時間，未來筆者或許會規劃)

最小單位 atom 元件
component / atom / *.html 為官方提供的設計請勿異動，若需要自定義元件請新增資料夾，譬如 component / {your namespace} / *.html
👽 自訂資料夾名<自訂元件名>(插槽)@class={css類別}|style={css樣式}|repeat=任意正整數|event={標籤事件屬性名:show(自定義css名稱)} <!--自定義css名稱前方需要有 ~ 波浪符號-->

atom<atom-file-name>(slot1:AAA,slot2:BBB)@class={bg-black text-white}|style={height:100px; width:100px}|repeat=3|event={onclick:show(~alert)}
atom 架構

👽 atom-file-name : <!--元件檔名-->

👽 @class={bg-black text-white}|style={height:100px; width:100px}|repeat=3|event={onclick:show(~alert)} 後綴屬性
目前規劃有 class style repeat，後續會增加其他功能
class <!-- 就是 html 標籤 class 屬性 -->
style <!-- 就是 html 標籤 style 屬性 -->
repeat <!-- 就是元件重複個數，適合製作假資料列表 -->
event <!-- 綁定事件用於 prototype 互動程式部分，下方將有 event 解釋 -->

簡單的程式互動用於原型設計 - 開關彈出畫面與跳轉網頁
event 介紹
👽 atom<btn>(點我跳出彈窗)@event={onclick:show(~alert)}  <!-- show 為開啟 ~alert 視窗 -->

onclick 為綁定事件名，支援所有 onXXXX 事件，譬如 onmouseenter / onmouseeleave

👽 若是設計一個 dialog 如下，記得加上 ~alert

atom<layout-v>@class={~alert fixed w-[100dvw] h-[100dvh] left-0 top-0 justify-center items-center bg-[#00000099]} <!-- 加上 ~alert -->
    atom<layout-v>@class={w-[320px] bg-[#CCC] border-1 rounded-lg overflow-hidden}
        atom<layout-v>@class={justify-center items-center h-[34px]}
            atom<txt-body>(Alert)
        atom<layout-v>@class={justify-center items-center bg-[#EEE] min-h-[100px]}
            atom<txt-body>(Hi, are you ok?)
        atom<layout-h>@class={justify-end items-center bg-[#EEE] p-1}
            atom<btn>(cancel)@class={min-w-[65px] h-[34px] rounded-sm mr-1 shadow-lg bg-[#DDD] hover:bg-[#CCC]}
            atom<btn>(ok)@class={min-w-[65px] h-[34px] rounded-sm shadow-lg bg-[#DDD] hover:bg-[#CCC]}|event={onclick:hide(~alert)}  <!-- 同理可以用 hide 關閉 ~alert 視窗 -->

👽 目前事件共有 3 個，分別是前述提到的 show / hide，還有一個是頁面跳轉 gotoPage，用法如下

atom<btn>(點我跳轉畫面)@event={onclick:gotoPage(page2.html)}
